Types of non-translatable content

When you plan the localization of your project, it is important to identify content that should be left untranslated. Here are a few examples of such content:

  • proper names, such as brand names
  • contact details
  • code blocks
  • UI text (if the UI is not localized)
  • legal text (may need to be handled separately)
Tip: You can use glossaries to identify content that must be left untranslated.

UI text

If your project contains references to UI text, it is important to consider whether the UI is translated and if so, into which languages. If the UI is not localized, you should instruct translators on how they should approach UI text. For example, you may want them to provide a translation in brackets.
